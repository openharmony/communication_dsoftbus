# 系统事件管理 (Event Management)

## 概述

系统事件管理模块是组网子系统与外部系统状态交互的核心桥梁。负责注册和分发 WiFi/BT 状态变化、屏幕/用户/账户状态、网络接口变更等系统事件，同时管理物理子网（WiFi IP、BT、SLE）的生命周期。模块采用观察者模式，允许多个组件注册同一事件的处理器。

**定位**：事件中枢，将系统状态变化转化为组网模块内部的响应动作。

## 核心功能

- **事件注册与分发**：基于观察者模式，支持同一事件注册多个处理器，线程安全的注册/注销
- **WiFi 状态监听**：连接/断开/开关等多种 WiFi 状态变化
- **BT 状态监听**：BLE/BR 开关、ACL 连接/断开
- **屏幕/用户/账户状态**：屏幕亮灭、锁定、用户前后台切换、账户变更、用户切换
- **网络接口管理**：IP 地址变化、网络接口状态变化
- **物理子网管理**：注册/注销/状态变更多种协议类型的物理子网
- **节点上下线通知**：通过 IPC 将设备上下线事件推送到 SDK 客户端
- **设备信息变更通知**：本地设备信息变更时通知相关组件
- **InfoKey 信息访问**：统一键值枚举提供设备信息的标准化访问方式

## 架构分层

| 层次 | 组件 | 职责 |
|------|------|------|
| 事件系统 | bus_center_event | 事件注册、分发、IPC 转发 |
| 事件监控 | lnn_event_monitor | 监听 Netlink/BT/WiFi/驱动等底层事件源 |
| 网络管理 | lnn/net_buscenter | WiFi IP 实现、BT 实现、物理子网管理 |
| 初始化监控 | lnn_init_monitor | 系统启动事件监控 |

**依赖**：Looper 消息机制、IPC 通知机制

**被依赖**：所有组网子模块（均通过事件系统获取状态变化通知）

## 对外接口

| 接口类别 | 说明 |
|---------|------|
| 事件注册/注销 | 注册和移除事件处理器 |
| WiFi 状态通知 | WiFi 连接/断开/开关状态变化通知 |
| BT 状态通知 | BT 开关、ACL 连接状态变化通知 |
| 屏幕状态通知 | 屏幕亮灭状态变化通知 |
| 节点状态通知 | 设备上下线、地址变更通知 |
| 设备信息通知 | 本地设备信息变更通知 |
| 网络状态通知 | 网络类型使能/禁用状态变化通知 |
| 物理子网管理 | 物理子网注册/注销/状态变更/全类型离线通知 |

## 设计约束与规则

| 规则 | 说明 |
|------|------|
| 线程安全 | 事件注册/分发使用 mutex 保护，handler 在锁外执行 |
| 防重复注册 | 同一 handler 对同一事件只能注册一次 |
| 异步通知 | 事件通知通过 Looper 异步投递，不阻塞调用方 |
| IPC 转发 | 节点上下线等事件需通过 IPC 推送到 SDK 客户端 |
| 处理器链表 | 每种事件类型可注册多个 handler，按注册顺序调用 |
| NetworkID 延迟更新 | NetworkID 变更使用延迟消息避免频繁更新 |
| 事件类型总数 | 约 70 种事件类型（系统监控 + 内部 LNN + SA 监控） |
