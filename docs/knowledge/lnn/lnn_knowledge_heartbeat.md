# 心跳保活 (Heartbeat)

## 概述

心跳模块通过周期性发送检测报文维持分布式网络中设备间的在线感知。模块区分主节点（Master）和普通节点（Normal）两种角色，主节点主动广播心跳，普通节点被动监听。支持 BLE、UDP、TCP、SLE 等多种通信介质，通过可调档位（GearMode）控制检测频率，在功耗和实时性之间取得平衡。

**定位**：设备在线感知引擎，心跳超时触发设备离线和退网流程。

## 核心功能

- **心跳发送与检测**：周期性发送心跳报文，检测远端设备存活状态
- **主从角色管理**：Master 节点主动广播，Normal 节点被动监听，支持动态切换
- **档位策略**：支持单次发送、固定周期、可调周期、直接发送等多种策略
- **多种介质支持**：BLE V0-V4、UDP、TCP Flush、SLE 共 7 种心跳类型
- **离线检测**：心跳超时后触发设备离线判定，驱动退网流程
- **屏幕状态联动**：屏幕关闭时加速离线检测
- **权重比较**：通过心跳报文携带权重信息辅助 Master 选举
- **中继心跳**：支持通过中间设备中继传递心跳

## 架构分层

| 层次 | 组件 | 职责 |
|------|------|------|
| 策略层 | lnn_heartbeat_strategy | 档位控制、策略选择、离线计时 |
| FSM 层 | lnn_heartbeat_fsm | 3 状态 FSM 驱动心跳生命周期 |
| 介质层 | lnn_heartbeat_medium_mgr | 适配 BLE/UDP/TCP/SLE 等通信介质 |
| 工具层 | lnn_heartbeat_utils | 类型转换、哈希生成、设备信息查询 |

**依赖**：状态机框架(infra)、事件系统(event)

**被依赖**：Net Builder（心跳超时触发退网）、事件系统（设备状态变更通知）

## 对外接口

| 接口类别 | 说明 |
|---------|------|
| 心跳启停 | 启动/停止心跳，支持按类型和策略启动 |
| 角色管理 | 设置/查询主节点或普通节点角色 |
| 档位控制 | 设置/获取心跳档位模式 |
| 离线计时 | 启动/停止设备离线计时（含屏幕变化场景） |
| 心跳使能 | 启用/禁用指定心跳类型 |
| 发送信息更新 | 更新心跳报文中携带的信息 |

## 设计约束与规则

| 规则 | 说明 |
|------|------|
| 心跳启动延迟 | 初始启动延迟 |
| 单次发送时长 | 每次心跳发送持续时长 |
| 设备离线时间 | 超时未收到心跳判定离线 |
| 通知延迟 | 离线后通知上层、主节点切换通知的延迟 |
| SLE 周期 | SLE 心跳周期和离线时间 |
| BLE 广播随机化 | 广播间隔随机化防碰撞 |
| FSM 消息驱动 | 所有状态切换通过 Looper 消息 |
| 介质可插拔 | 通过注册 MediumMgr 支持新介质 |
